Testing ARIA role=timer

Published: March 2, 2022

A role=timer is a live region with an implicit aria-live=off.

An aria-live=off is not meant to announce dynamic changes "unless the user is currently focused on that region".

The following test has a role=timer with a tabindex=0 to allow for testing with Virtual Cursor "focus" as well as keyboard focus.

role="timer" in the DOM on page load

results
  • macOS 11.6 VoiceOver with Safari 15.1: updates not announced when focused with VC or keyboard focus. Timer role announced.
  • JAWS 2022.2110.60 with Edge & Chrome 98 updates not announced when focused with VC or keyboard focus. Timer role not announced.
  • JAWS 2022.2110.60 with Firefox 97.0.1: no announcement of content made if focused with Tab key. VC focus announces current value but not dynamic updates. Timer role not announced.
  • NVDA 2021.3.3 with Edge & Chrome 98: updates not announced when focused with VC or keyboard focus. Timer role not announced.
  • NVDA 2021.3.3 with Firefox 97.0.1: updates not announced when focused with VC or keyboard focus. Timer role not announced.
  • Narrator with Win10 and Edge 98: updates not announced when focused with VC or keyboard focus. Timer role announced.
  • Android 12 TalkBack with Firefox 95.2: updates announced while VC focus remains on element. Timer role not announced.
  • Android 12 TalkBack with Chrome 96: updates not announced when focused with VC. Timer role announced.
  • iOS 15.3.1 VoiceOver with Safari: no announcement of dynamic content. Is difficult to swipe to the dynamically updating content. VO focus will often skip over the countdown timer (nothing to do with the role. iOS VO just doesn't seem to like moving focus to content that updates so frequently). Timer role not announced.
